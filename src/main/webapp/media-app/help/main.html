About the software
The software implements an integrated solution for:
  • editing, managing editorial metatda data
  • receiving the medata from multiple sources and integrate them (ESB)
  • receives and manage media file such as video and images
  • automatically link the media files to the metadata
  • auto-resize the images and auto-manage the files
  • push the metadata &media files to the brightcove
  • Hierchical management for metata, images, and tags
  • Automatic rules for advertisement
  • the files are managed in s3, and auto pickup of the changes in s3, this means integration with other file management tools
Software components

The running environment consists of mysql, Mule ESB and nginx load balancer, prepard with docker: https://github.com/boxmediapp/bdocker

The server component is an Mule ESB project: https://github.com/boxmediapp/box-mule-api

The client UI component is a single page HTML5 app: https://github.com/boxmediapp/box-media-client-app

Q: After I have cleared out the ingest source field or the image file name field, and clicked on "Save", the value in field has appeared again, is it a bug?

A: As long as there is matching video/image file name in the s3 bucket for the episode, the field will always be filled in. If you think the video or image file is not correct, the file should be renamed.

Q: Does Programme get imported from Schedule?

A: Yes.

Q: Does the PRogramme Group get automatically created or is this manually added?

A: The app creates the programme collection with the same title as the programme or will be added to the "Default Series Group". You need to modify the programme collection title to the correct one, the programme that has the same programme collection will be joined together into the same collection automatically.

Q: How do we reconcile against existing data in Brightcove against Box Media App (will it over-write)?

A: Whatever entered in the Box Media app, if not left blank, will overwrite the data in the Brighcove. For example, if you wish the images in the Brightcove should stay, then you should not specify the image in the Box Media app. If you wish the Synopsis in the Brightcove to be kept as it is then you should left synopsis field blank in the Box Media app. But ideally it is not recommended to scatter around the master data all around in different systems, it is a maintenance nightmare you know. There is a button "Import images from brightcove" in the Box Media app, so you can import images from the Brightcove, there will soon a button called "Import everything" so you can import everything from brightcove.

Q: If Availability window is left blank, does this mean it is available indefinitely?
